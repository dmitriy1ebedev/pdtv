## Описание

`pdtv` — bash-утилита для обработки входящих вложений и формирования **ПДТВ** —
текстового подтверждения. ПДТВ перечисляет принятые файлы с их размерами и
отправляется обратно отправителю как свидетельство, что письмо дошло до адресата.

ZIP обрабатывается первым: несколько вложений часто приходят одной кнопкой «скачать
всё» в виде ZIP, а одиночное вложение — как есть. Поэтому ZIP распаковывается, и в
ПДТВ попадают сами файлы, а не архив.

Скрипт рассчитан на локальную работу в Astra Linux 1.7+ и Alt Linux 10.4+,
поддерживает CLI для автоматизации и графический режим через Zenity (включая
запуск по ярлыку рабочего стола).

## Возможности

- **модульная архитектура** — этапы M2–M7, каждый можно включить/выключить;
- распаковка ZIP из «Входящих», упаковка бланков (doc/docx/txt/odt);
- распаковка архивов `zip/7z/rar/tar/gz/bz2/xz`, включая **«архив в архиве»**
  (рекурсивно); архивы распознаются и по содержимому (сигнатуре), а не только по
  расширению — это ловит архив под «чужим» расширением;
- **офисные документы не «потрошатся»**: `docx/odt/xlsx/pptx/…` — это ZIP-контейнеры
  (сигнатура `PK`), но они распознаются как документы, а не как архивы, и при
  распаковке не разбираются на внутренний xml-«скелет»;
- месячная иерархия каталогов «`01 Январь/<дата>`»;
- формирование текстового ПДТВ с перечнем файлов и **размерами в байтах**;
- при отсутствии базовых каталогов (`Входящие`, `ПДТВ`, `00_Отработанные`) скрипт
  **предлагает их создать**;
- несколько одноимённых ПДТВ за день не перетирают друг друга (нумерованные
  резервные копии);
- подпись ПДТВ переменной `SIGNATURE` с подстановкой `{фио}` (ФИО берётся из GECOS
  или задаётся вручную);
- цветной интерфейс с итоговой сводкой (`USE_COLOR=auto`, уважает `NO_COLOR` и
  не-TTY);
- графический режим Zenity: окно настроек открывается при запуске **без аргументов**
  в графической сессии либо по ключу `--gui`; запуск по ярлыку и создание ярлыка
  в домашнем каталоге;
- **запоминание настроек окна**: рабочая папка и ФИО дежурного сохраняются в
  `~/.config/pdtv/config` и подставляются при следующем запуске (ключи CLI важнее
  сохранённого);
- **автооткрытие результата**: по окончании прогона открываются текстовые документы
  (`doc/docx/txt/odt`), `pdf` и таблицы (`ods/xlsx`); открытие выполняется **до**
  раздачи прав (`chmod`), чтобы окна появлялись сразу;
- режимы `--dry-run` (прогон без изменений на диске) и `--verbose`.

## Модули

| Модуль | Имя | Назначение |
|---|---|---|
| M1 | — | подготовка окружения (каталоги периода, перенос остатков); предупреждает о файлах во «Входящих» без расширения / с незнакомым расширением — их обработчик не берёт |
| M2 | `blanks` | упаковывает бланки (doc/docx/txt/odt) в ZIP |
| M3 | `unzip` | распаковывает внешний ZIP из «Входящих» в «Загрузку» |
| M4 | `pdtv` | формирует ПДТВ — перечень **полученных** файлов (как пришли) |
| M5 | `extract` | распаковывает архивы, включая «архив в архиве» (по содержимому) |
| M6 | `cleanup` | раскладывает результат по уровню вложенности, очищает «Загрузку» |
| M7 | `open` | открывает результат (Отработанные/<дата> и Входящие): doc/docx/txt/odt, pdf, ods/xlsx |

Порядок выполнения: M1 → M2 → M3 → **M4 (перечень ПДТВ по полученным файлам)** →
глубокая распаковка вложенного (M5, до стабилизации) → M6 → M7.

### Куда попадают файлы

Раскладка зависит от **уровня вложенности**, а не от типа файла:

- **скачан один файл** — он пакуется в ZIP, архив кладётся в папку дня
  `00_Отработанные/<месяц>/<дата>/`, сам файл лежит рядом и сразу открывается;
- **скачан архив** — то, что лежит прямо в нём (первый уровень), кладётся в папку дня
  рядом с архивом и открывается;
- **архив внутри архива** — всё его содержимое (второй уровень и глубже) попадает
  в корень «Входящих».

Одноимённые файлы разных уровней не конфликтуют: верхний уедет в папку дня, вложенный
останется во «Входящих». Поведение отключается настройкой `LEVEL1_TO_DONE=false` —
тогда во «Входящих» остаётся всё.
Поэтому в ПДТВ попадают именно полученные файлы (содержимое внешнего ZIP, принятые
архивы), а итоговое содержимое из цепочки «архив → архив → …» вытаскивается в корень
«Входящих».

Модули отключаются постоянно (в блоке «МОДУЛИ» в шапке скрипта, `ENABLE_*=false`)
или разово из CLI: `--only blanks,pdtv`, `--skip blanks,chmod`, `--no-extract` и т.п.
Список модулей и их статус — `--list-modules`.

## Установка

```bash
chmod +x pdtv.sh
./pdtv.sh --help
```

Для установки в системный PATH:

```bash
sudo install -m 755 pdtv.sh /usr/local/bin/pdtv
```

## Зависимости

Обязательные:

- `bash` (>= 4, НЕ dash);
- `find`, `sed`, `sort`, `paste`, `grep`, `tr`, `wc` и стандартный набор coreutils.

Опциональные (при отсутствии соответствующий этап пропускается):

- `zip`, `unzip` — упаковка/распаковка ZIP;
- `7z` (p7zip) — архивы 7z/rar/zip, запасной экстрактор;
- `tar`, `gzip`, `bzip2`, `xz` — соответствующие архивы;
- `getent` — чтение ФИО из GECOS;
- `zenity` — графический запрос дежурного при запуске по ярлыку;
- `xdg-open` — открытие файлов в графической сессии;
- `numfmt` — человекочитаемый размер.

Установка пакетов в Debian-подобных системах:

```bash
sudo apt install zip unzip p7zip-full tar gzip bzip2 xz-utils zenity xdg-utils coreutils
```

## Быстрый старт

```bash
./pdtv.sh                          # полный прогон
./pdtv.sh --dry-run --verbose      # показать действия БЕЗ изменений на диске
./pdtv.sh -d 31.12.2026            # задать рабочую дату
./pdtv.sh --root /mnt/docs         # другой корневой каталог
./pdtv.sh --officer "Иванов И.И."  # задать ФИО дежурного
./pdtv.sh --only pdtv,extract      # выполнить только нужные модули
./pdtv.sh --skip blanks,chmod      # пропустить часть модулей
./pdtv.sh --list-modules           # список модулей и их статус
```

## Параметры CLI

| Параметр | Описание |
|---|---|
| `-d`, `--date ДАТА` | рабочая дата `ДД.ММ.ГГГГ` (по умолчанию — сегодня) |
| `--root ПУТЬ` | корневой каталог (по умолчанию `/DOCS/Общая`) |
| `--dry-run` | показать действия без изменений на диске |
| `--verbose` | подробный лог |
| `--quiet` | только предупреждения, ошибки и итоговая сводка |
| `--pause` / `--no-pause` | ждать / не ждать Enter в конце |
| `--color` / `--no-color` | принудительно вкл/выкл цвет (учитывается `NO_COLOR`) |
| `--month` / `--no-month` | уровень месяца в каталогах: «`01 Январь/<дата>`» |
| `--officer ФИО` | задать ФИО дежурного вручную (перебивает GECOS) |
| `--max-depth N` | глубина распаковки «архив в архиве» (по умолчанию 5) |
| `--keep-archives` | не удалять архивы, оставшиеся после предела вложенности (`--max-depth`), а переносить их в «Отработанные» |
| `--gui` / `--no-gui` | принудительно открыть / никогда не открывать окно Zenity |
| `--make-shortcut` | создать ярлык и выйти; кладётся на рабочий стол (путь берётся у `xdg-user-dir DESKTOP`) и копией в `$HOME`; в имени и подписи — ФИО дежурного (`--officer`), у каждого свой ярлык (чужой не затирается) |
| `--diagnostic` | собрать диагностический отчёт в `$HOME/pdtv-diagnostic-*.txt` и выйти (ничего не меняет): окружение, инструменты и их версии, графическая сессия, конфиг, дерево рабочей папки с типом файлов, само-тест ярлыка, захват `--dry-run --verbose`. Для разбора проблем на удалённых АРМ по логам |
| `--only СПИСОК` | выполнить ТОЛЬКО эти модули (через запятую) |
| `--skip СПИСОК` | пропустить эти модули (через запятую) |
| `--list-modules` | показать модули и их статус, выйти |
| `--no-blanks` … `--no-open` | отключить отдельный модуль |
| `-h`, `--help` | справка |
| `-v`, `--version` | версия |

Имена модулей: `blanks unzip pdtv extract cleanup chmod open`.

## Настройка под среду

Параметры по умолчанию задаются в шапке `pdtv.sh`:

- блок «МОДУЛИ» — включение/выключение этапов (`ENABLE_*`);
- переменная `SIGNATURE` — шаблон подписи ПДТВ с `{фио}`;
- корневой каталог, политика месячной иерархии, поведение цвета (`USE_COLOR`).

Целевая среда — Astra Linux. Ярлык создаётся в домашнем каталоге (`~`): на части
Astra путь до рабочего стола плавает и ярлык «пропадает», а `~` стабилен — оттуда
его можно самому скопировать на рабочий стол.

Параллельный запуск в одной рабочей папке (два дежурных кликнули свои ярлыки
одновременно) блокируется замком `flock`: второй запуск сообщает «обработка
уже идёт» и не вмешивается.
