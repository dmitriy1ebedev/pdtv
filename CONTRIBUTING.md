# Contributing

Спасибо за интерес к проекту `pdtv`.

## Как вносить изменения

1. Создайте ветку от актуального `master`.
2. Вносите правки только в соответствующие файлы.
3. Проверяйте синтаксис (`bash -n pdtv.sh`) и `shellcheck -S warning pdtv.sh`.
4. Прогоняйте сценарий на тестовом каталоге с `--dry-run --verbose` перед коммитом.
5. Описывайте результат в сообщении коммита и в pull request.

## Что особенно важно

- сохраняйте совместимость с Bash 4+ (НЕ dash);
- не добавляйте внешние зависимости без причины — недостающие инструменты должны приводить к пропуску этапа, а не к падению;
- не изменяйте логику модулей без необходимости и не ломайте управление модулями (`--only`/`--skip`/`--no-*`);
- держите код чистым по `shellcheck` (уровень warning);
- при изменении лицензионных файлов синхронизируйте `LICENSE.txt`, `NOTICE` и `README.md`.
