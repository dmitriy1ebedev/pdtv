# Changelog

## 1.2

### Исправлено
- **Документы из архива больше не открываются с ошибкой «файл не существует».**
  Каталог `! Отработанные` переименован в `00_Отработанные`. Восклицательный знак
  и пробел в имени папки разрывали путь внутри системной открывалки (старый
  `xdg-open` на Astra подставляет путь в команду без кавычек), и файл «пропадал».
  Существующий каталог переименовывается автоматически при первом запуске,
  вместе со всем содержимым.
- **Файлы из присланных архивов остаются во «Входящих».** Раньше документ,
  лежавший в архиве рядом с вложенным архивом (`письмо.txt` + `вложенный.zip`),
  уезжал в папку с датой, а из вложенного архива во «Входящие» попадал только
  PDF. Теперь в архив дня убираются лишь бланки, которые скрипт упаковал сам.
- **Одноимённые файлы из разных архивов больше не затирают друг друга.**
  Если в двух архивах лежал `акт.txt`, один документ пропадал безвозвратно.
  Теперь архивы распаковываются через временный каталог, а второй экземпляр
  сохраняется рядом под именем `акт.txt.~1~`.
- **Файл замка не мешает рядовым операторам.** Замок, созданный администратором,
  был недоступен обычному пользователю на запись. Теперь права выставляются
  корректно, а при отказе скрипт продолжает работу с понятным предупреждением
  вместо тихого сбоя.

### Добавлено
- **Надёжное открытие файлов.** Скрипт по очереди пробует `gio open`, `fly-open`
  и `xdg-open`, пока файл не откроется, и предупреждает, если не смог. В режиме
  `--verbose` видно, что именно ответила открывалка.
- **Настройки** (в шапке скрипта): `DONE_DIR_NAME` — имя каталога-архива,
  `MIGRATE_DONE_DIR` — автоматический переезд со старого имени,
  `MOVE_M2_BLANKS_TO_DONE` — убирать ли упакованные бланки в архив дня,
  `OPEN_TOOL` — чем открывать файлы.

## 1.1

### Исправлено
- **Ярлыки для нескольких дежурных.** Раньше все ярлыки записывались в один
  файл `pdtv.desktop` — созданный вторым дежурным ярлык затирал ярлык первого.
  Теперь у каждого дежурного свой файл (в имени и подписи ярлыка — его ФИО),
  ярлыки живут рядом и не мешают друг другу. Если тот же дежурный создаёт
  ярлык повторно — обновляется только его ярлык.
- **Ярлык кладётся на видимый рабочий стол.** На Astra ярлык теперь всегда
  попадает в `~/Desktops/Desktop1` — тот стол, который реально показан на
  экране. Раньше система могла подсунуть каталог `~/Desktop`, которого на
  экране нет, и ярлык «исчезал».

### Добавлено
- **Защита от двойного запуска.** Если обработка в этой рабочей папке уже
  идёт (например, второй дежурный кликнул свой ярлык), новый запуск честно
  сообщает об этом и не вмешивается — файлы не перемешиваются.
